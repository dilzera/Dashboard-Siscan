Gere um sistema de Dashboards Web Interativos (MVP) utilizando a biblioteca Dash em Python, ideal para consumir dados estruturados de um banco de dados PostgreSQL (como o Supabase). O sistema deve ser focado em usabilidade (UX) e facilidade de manutenção (UI).1. Requisitos de Arquitetura e Dados (Backend)Fonte de Dados: Conexão direta a um banco de dados PostgreSQL (simulando a sua base "Dados Consolidados SISCAN").Camada de Conexão: Uso da biblioteca SQLAlchemy para gerenciar a conexão e o carregamento dos dados, garantindo separação clara entre a lógica de aplicação e o banco.Atualização de Dados: O sistema deve incluir um mecanismo de cache ou atualização periódica (por exemplo, a cada 5 minutos ou sob demanda via botão) para garantir que os dashboards exibam os dados mais recentes do PostgreSQL.2. Requisitos de Frontend e Interatividade (UX/UI)Layout: Crie um layout responsivo (adaptável a desktop e mobile) utilizando Dash/Bootstrap Components (ou equivalente). O design deve ser limpo e profissional, com paleta de cores focada em alto contraste para clareza dos dados críticos (BI-RADS 4/5).Filtros Dinâmicos (Controles de Usuário):Implemente filtros interativos (dropdowns e sliders) para os seguintes campos:Ano: Seleção do Ano da Solicitação/Realização.Unidade de Saúde: Dropdown com busca textual.Status de Conformidade: Filtro por "Dentro do Prazo" vs. "Fora do Prazo".Gráficos Interativos: Todos os gráficos devem ser gerados com Plotly Express para permitir zoom, hover (detalhes ao passar o mouse) e seleção de categorias.3. Chamada de Ação Final"O código deve ser modular (separando layout.py, callbacks.py e data_loader.py). Forneça as classes e funções principais com placeholders para a lógica SQL de extração de dados e detalhe como o sistema permite alterar os filtros para que o usuário final possa realizar análises ad-hoc (drill-down)."
